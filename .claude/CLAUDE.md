# グローバル設定

このドキュメントは、Claude Codeのグローバル設定のエントリーポイントである。

## 前提条件

以下のMCPサーバーとCLIツールのセットアップが必要である。

### 必須MCP
- **Serena MCP**: コード解析
- **Context7 MCP**: ライブラリ情報

### 必須CLI
- **gemini-cli**: Web検索
- **agent-browser**: ブラウザ操作（基本）

### 推奨MCP
- **Playwright MCP**: ブラウザ制御（高度）

**注**: 未セットアップの依存は、実際に必要になった段でインストール方法を提案する。詳細は `rules/prerequisites.md` を参照。

## 最重要ルール

以下のルールは**必ず**守ること。

1. **思考言語**: 英語で思考し、日本語で応答すること
2. **取り組み方（理解・調査・判断）は「丁寧・細かく・正確」**: 急がず時間をかけ、手順を飛ばさない【丁寧】／粒度を細かく取り、細部・例外・個別ケースまで見る【細かく】／推測で済ませず事実・実物に照合して確かめる【正確】。三つを欠けば雑——拙速・大雑把・思い込みに落ちる。**次項「出力」の簡潔さをここに持ち込まない**（拙速な削除・即断は雑）。
3. **出力（成果物・応答）は「本質・シンプル・端的」**: 核（原則・原因・構造・真の課題）を捉える【本質】／答え・構造・表現は最小限に保ち複雑さを足さない【シンプル】／要点を短く言い切り、修飾・前置き・具体の羅列で核を覆わない【端的】。三つは同根——核を外すほど具体で埋め合わせて冗長になり複雑になる。出力前の自問は `rules/conduct-rules.md` の出力ゲート。

## モデル選択ガイドライン

タスクの性質に応じて適切なモデルを使用する。

- **Opus**: 要件定義、アーキテクチャ設計、複雑な問題解決
- **Sonnet**: 実装、リファクタリング、テスト作成（デフォルト）
- **Haiku**: 簡単な調査、ファイル検索、軽量タスク

**注**: 現状、自動切り替えはできないため、Plan モードや Task ツールで明示的に指定する。

## 情報収集の方法

### Web検索

組み込みの WebSearch に加え、Gemini CLI 経由の検索も任意で使える。

```bash
gemini --skip-trust --prompt 'WebSearch: <検索クエリ>'
```

### ライブラリ・フレームワークの情報取得

**Context7 MCP**を使用して最新情報の取得を試みる。

### コード解析

**Serena MCP**を使用する。以下の優先順位でツールを使用する。

1. **シンボル/関数検索**: 常に `find_symbol` を最初に試す
2. **テキスト/文字列検索**: `search_for_pattern` を使用

## 壁打ち（多視点レビュー）の活用

実行者は Claude 本体（タスク分解・実装・リファクタリング・ファイル操作をステップごとに進める）。不確実な判断・技術検証で第三者視点が要る時は、**外部モデルでなく批判役の subagent（Agent tool）を立てて設計・結論を叩く**（Claude 内で完結し、ユーザーを介さず agent 間で検証できる）。

発動する状況: ①実装前の設計レビュー（考慮漏れ・複数案比較） ②技術的制約・実装可能性の確認 ③エラー解析 ④実装後のコードレビュー（性能・セキュリティ・品質）。最新情報の確認は WebSearch / Context7 / `tech-research` subagent を使う。

※ 専用の批判役エージェントは未整備。当面は汎用 subagent を批判役として起動する。

## ブラウザ操作ツールの使い分け

### agent-browser（基本）

- 基本的なブラウザ操作（ページを開く、クリック、入力）
- トークン効率が良い
- **デフォルトのブラウザ操作ツールとして使用**

### Playwright MCP

- スクリーンショット取得、ネットワーク監視、複雑なページ遷移のデバッグ
- **agent-browserでできない操作に使用**
