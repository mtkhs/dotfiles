# セッション管理

Claudeセッションの効率的な管理方法を定義する。

## 進捗の定期保存

長時間作業では、セッション継続失敗に備えて定期的に進捗をmdファイルに保存する。

### 保存タイミング

- 重要な設計決定をした時
- 大きな実装を完了した時
- 1時間以上継続して作業している時
- 複雑な調査結果をまとめた時

### 保存内容

```markdown
# 進捗レポート - [日時]

## 完了
- [タスク1]

## 進行中
- [タスク名]: [状況]

## 次のステップ
1. [次にやること]

## 決定事項
- **[決定]**: [理由]

## 問題点
- [問題]: [詳細]
```

ファイル名: `progress/YYYY-MM-DD-HHMM.md`（例: `progress/2026-01-25-1630.md`）

## コンテキスト管理

### リセットのタイミング

以下の場合はコンテキストをリセット（`/clear`）する。

- 全く異なるタスクに移る時
- エラーが続き、試行錯誤の履歴が溜まった時
- 大量のコード読み込み後、別の作業に移る時
- セッションが2時間以上継続している時

## セッションの再開

### 進捗ファイルからの再開

1. 最新の進捗ファイルを読む
2. 「次のステップ」から作業を継続
3. 「決定事項」を踏まえて進める

### 長期中断後の再開

1. プロジェクト全体像を再確認
2. 最新の進捗ファイルを確認
3. 変更されたファイルを確認（`git status`等）
4. 必要に応じてSerenaのonboarding確認
