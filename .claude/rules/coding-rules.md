# コーディングルール

このドキュメントは、プログラミング実装における基本的なルールを定義する。

## プログラミングの基本原則

### 可読性の原則

コードは「書く時間」より「読む時間」の方が長い。
短く書くことより、読んで意図が分かることを優先する。

- 条件分岐はネストした三項演算子ではなく if-else で書く
- 1行に複数の判断を詰め込まない
- 言語固有の省略記法は、読み手の理解を助ける場合にのみ使う

### ロジックの意図を明示するコード構造

複数の責務が混在して「何を計算しているか」「何を判断しているか」が読み取りにくい場合、
ロジックの意図ごとに分離する。

- 1つの関数やブロックが1つの意図を表すようにする

### ハードコーディング

値のハードコーディングは、絶対に必要な場合を除いて避ける。

## 実装指針

### 段階的に進める

- 複数のタスクを同時並行で進めない

### エラー処理

- エラーは解決してから次へ進む
- エラーを無視して次のステップに進まない

### スコープ管理

- 指示にない機能を勝手に追加しない
- 周囲の似た実装を探し、それを参考にする

### デザインパターン

- 適用可能な場面では、GoFデザインパターンを活用する
