# セッション継続管理

このドキュメントは、セッションをまたいで作業を継続するための方法を定義する。

## セッション再開

基本的に `claude -c` で前回の会話を継続する。
長期中断後の新規セッションでは、Memory MCP からコンテキストを復元する。

## Memory MCP の活用

プロジェクトの決定事項・設計の根拠を知識グラフとして保存し、セッションをまたいで参照する。

### 保存するもの

- アーキテクチャ・設計の決定とその理由
- 技術スタック・ライブラリの選定理由
- プロジェクトの重要な制約・前提

### 保存すべきでないもの

- 認証情報・APIキー
- セッション内の一時的な作業状態

### 保存タイミング

重要な決定をしたタイミングで即座に記録する。

### 新規セッション開始時の読み出し

`claude -c` でない場合は `mcp__memory__search_nodes` でプロジェクト名を検索し、コンテキストを復元する。

## コンテキスト管理

以下の場合は `/clear` でコンテキストをリセットする。

- 全く異なるタスクに移る時
- エラーが続き、試行錯誤の履歴が溜まった時
- 大量のコード読み込み後、別の作業に移る時
